1. ALOHA вАУ Pure & Slotted

In random access protocols, no station is superior. Any station transmits when it has data. Collisions are possible and handled by retransmission.

Pure ALOHA

2. CSMA / CSMA-CD / CSMA-CA

CSMA: "Listen before talk." Sense carrier before transmitting. Reduces (not eliminates) collisions.

  • 1-persistent: If busy keep listening; transmit immediately when idle. High collision probability.
  • Non-persistent: If busy wait random time, re-sense. Lower load, higher delay.
  • p-persistent: If idle, transmit with probability p, defer with 1вИТp.

CSMA/CD вАУ Collision Detection (Wired)

While transmitting, keep listening. If collision detected вЖТ send JAM signal (48 bits) вЖТ stop вЖТ Binary Exponential Back-off вЖТ retry. Used in Ethernet (802.3).

A вФАвФАвФАвФАвФАвФАвФАвФАвФАвФАвЦЇcollisionвЧДвФАвФАвФАвФАвФАвФАвФАвФАвФАвФА B detects energy spike вЖТ sends 32-bit JAM both abort вЖТ back-off вЖТ retry Back-off after k-th collision: Pick r randomly from {0, 1, ..., 2^k вИТ 1}, wait r √Ч Tslot Max k = 10 for back-off; discard after 16 failures.

CSMA/CA вАУ Collision Avoidance (Wireless)

Cannot detect collision while transmitting (half-duplex). Strategy: avoid before transmit. IEEE 802.11 Wi-Fi.

  • Wait for IFS (Inter-Frame Space) + random back-off before transmitting.
  • Optional RTS/CTS handshake to reserve channel.
  • Mandatory ACK for every unicast frame.
рЯМН Real-life: Ethernet hub networks use CSMA/CD. Your home Wi-Fi (802.11) uses CSMA/CA.

3. Controlled Access вАУ Polling & Token Passing

Polling

A primary (master) station polls each secondary in a round-robin: "Do you have data?" Only polled station can transmit. No collisions.

рЯМН Real-life: Bluetooth piconet (master polls slaves). Mainframe-to-terminal legacy systems.

11. Routing Algorithms вАУ Dijkstra, Flooding, Distance Vector, Link State

Shortest Path (Dijkstra's Algorithm) вАУ used in Link State

Each router has complete topology. Runs Dijkstra to compute shortest path to every destination.

рЯУР Dijkstra's Update Rule

d[v] = min( d[v], d[u] + w(u,v) )

where u = visited node, v = unvisited neighbor, w = link cost

Start: d[source]=0, all others=вИЮ. Always pick minimum unvisited node next.

Flooding

Every packet sent out on all interfaces except incoming one. Guarantees delivery. Controlled by hop count or sequence numbers to avoid infinite loops.

Distance Vector Routing (Bellman-Ford)

Each router knows only its neighbors' costs. Shares its distance vector with direct neighbors. Iteratively converges.

рЯУР Bellman-Ford Equation

d(x,y) = min over all neighbors v [ c(x,v) + d(v,y) ]

x = current router, y = destination, v = neighbor of x, c = link cost to v, d(v,y) = v's distance to y

Phase 1: Each router floods LSA (Link State Advertisement) to all routers вАФ every router gets full topology.

16. Sliding Window Protocols вАУ Stop-and-Wait, GBN, SR

These protocols control how many unacknowledged frames a sender can have "in flight" at once.

Stop-and-Wait

Send one frame вЖТ wait for ACK вЖТ send next. Window size = 1. Very inefficient on high-delay links.

Go-Back-N (GBN)

Sender can have up to W frames outstanding. On error, receiver discards all frames after error and sender retransmits from error frame onward.

GBN Window (W=4): Sent+ACKed | 1 | 2 | 3 | 4 | (next to send) If frame 2 errors вЖТ discard 2,3,4 вЖТ retransmit 2,3,4 Receiver buffer = 1 (only accepts in-order)

Selective Repeat (SR)

Sender can have W frames outstanding. On error, receiver buffers out-of-order frames. Only the errored frame is retransmitted.

SR Window (W=4): Frame 2 errors вЖТ receiver buffers 3,4 вЖТ only 2 is retransmitted Sender buffer = W (keeps copies) | Receiver buffer = W
